## 设计理念

本工具的目标并非替代微信读书的阅读体验，更无规避版权和付费目的。如果仅为了阅读，建议直接用微信读书 App 。

**核心场景是：让 AI 能"读"微信读书里的书。**

导出结果为 Markdown 格式，对 LLM 友好，可直接作为 AI 分析、摘要、知识提取等任务的输入。配合 各种AI Agent 的自动化能力，可实现批量书籍分析、跨书知识串联等高级工作流。

不建议导出长篇小说（字数多，导出时间长）；不建议导出漫画、绘本或纯图片书籍（技术原理限制，导出内容不完整）

### 架构定位

```
微信读书 ──→ 导出工具 ──→ Markdown 文件 ──→ AI 分析
                ↑                              ↑
         应用版：一键傻瓜式              Skill版：按章节、可定制
         后台静默处理                   专属 Agent 异步导出
```

两种版本覆盖不同场景，技术路线（Canvas Hook 逐页捕获）决定了导出速度，但可后台运行，不影响用户做其他事情。

## 两种版本

| 维度 | 应用版（App） | Skill 版 |
|------|-------------|---------|
| 入口 | `python weread_exporter.py` | AI Agent 加载 Skill |
| 目标用户 | 人类操作，一键导出 | AI 代理，自动化工作流 |
| 交互 | 选书 → 全本自动导出 | 导引树 → 选章节范围 → 导出后菜单 |
| 灵活性 | 低（整本导出，无选项） | 高（按章节、可调范围） |
| 章节拆分 | 不拆分（一个文件一本书） | 按一级标题分文件 |
| 适用场景 | 快速导出整本书给 AI 分析 | 需要精确控制章节范围的复杂任务 |
| 后台运行 | ✅ 可后台静默处理 | ✅ 可单独开 Agent 专职导出 |

> Skill 版包含完整的 TOC 层级分类算法、导引树展示、章节范围选择、边界页处理等功能。
> 如需 AI 按需调整导出策略，使用 Skill 版并让 Agent 自行决策。

## 工作原理

### 核心技术

微信读书 Web 版将文字渲染在 **HTML5 Canvas** 上，而非传统 DOM 中，因此无法直接解析 HTML 提取文字。

本工具通过 **Hook Canvas 2D 的 `fillText` 方法**，拦截每次文字绘制调用，捕获文字内容及其屏幕坐标，再按坐标重组为自然阅读顺序（左栏行 → 右栏行）。

## 安装方法

### 环境要求

| 组件 | 要求 |
|------|------|
| 操作系统 | Windows Subsystem for Linux 2（WSL2），建议 Ubuntu 22.04+ |
| Python | 3.10+ |
| 浏览器 | Chromium（由 Playwright 自动安装） |
| 依赖 | `playwright` Python 包 |

> ⚠️ 暂不支持 macOS / 原生 Windows，因依赖 Chromium + WSLg 图形栈弹出登录窗口。

### 安装步骤

```bash
# 1. 进入项目目录
cd weread-canvas-exporter

# 2. （推荐）创建虚拟环境，避免污染系统 Python
python3 -m venv venv
source venv/bin/activate

# 3. 安装 Python 依赖 + Chromium 浏览器
pip install playwright
python -m playwright install chromium

# 4. 验证安装
python -c "from playwright.sync_api import sync_playwright; p=sync_playwright().start(); print('✅ 安装成功'); p.stop()"
```

或运行一键安装脚本 `./install.sh`。

## 使用方法（应用版）

### 基本命令

```bash
cd weread-canvas-exporter
source venv/bin/activate    # 如果用了虚拟环境
python weread_exporter.py
```

### 全流程

```
① 运行 python weread_exporter.py
② 无 cookie → 弹出浏览器，微信扫码登录（无超时，关窗即取消）
③ 登录成功 → 浏览器自动关闭，进入后台无头模式
④ 显示书架列表 ────── 输入编号选书
⑤ 检测是否付费书 ───── [Y]导出试读 / [N]回书架
⑥ 自动开始导出 ────── 实时显示翻页进度
⑦ 全书完 / 付费墙 ──── 保存 → [1]继续导出 / [2]退出
```

### 书架操作

| 操作 | 说明 |
|------|------|
| 输入 `1`、`2`... | 选择对应编号的书籍 |
| 按 `Enter`（空输入） | 翻到下一页（每页 20 本，循环） |
| 输入 `q` | 退出程序 |

### 付费书处理

检测到付费书时：

```
╔══════════════════════════════════════╗
║  本书需要付费会员才能阅读完整内容    ║
╚══════════════════════════════════════╝
是否导出试读部分？[Y/N]:
```

- `Y`：从第一页开始导出，翻到付费墙自动停
- `N`：返回书架

### 后台运行

长书的导出可后台静默处理，不影响做其他事情：

```bash
nohup python weread_exporter.py > export.log 2>&1 &
tail -f export.log    # 查看实时进度
```

也可通过 AI Agent 的 `terminal(background=true)` 在后台运行。

## 导出流程详解

### 流程图

```
选书 → 打开阅读器
  ├─ 关闭书友想法
  ├─ 注入 Canvas Hook
  ├─ Phase 1 付费检测 ─── .need_pay_mask 是否存在？
  │    ├── 否 → 免费书，正常导出
  │    └── 是 → 询问用户 → [Y] / [N]
  ├─ 导航到版权信息页
  ├─ ArrowRight 逐页翻页 + 捕获文字
  │    ├─ Phase 2 付费检测 ── .need_pay_mask 是否可见？
  │    ├─ 空页跳过（连续8页 → 全书完）
  │    └─ 内容相同检测（连续3页 → 全书完）
  └─ 写入 Markdown 文件
```

## 退出机制

| 场景 | 操作 | 效果 |
|------|------|------|
| 导出过程中想停 | **Ctrl+C** | 已导出的内容自动保存，回书架 |
| 选书时想退出 | 输入 `q` | 退出程序 |
| 导出完成后 | 选 `[2] 退出程序` | 退出程序 |
| 付费书提示 | 选 `N` | 回书架，不导出 |

按 Ctrl+C 时的表现：

```
^C
[12:34:56] 用户中断导出
  💾 已保存已导出的 87 页内容至: output/书名/书名.md
```

> 每 50 页自动写入一次磁盘，最多丢失 50 页。
> 如果不想保留已导出的内容，删除对应的 `.md` 文件即可。

## 常见问题

### WSL 卡死 / 关闭怎么办？

Chromium + WSLg 图形栈在长时间运行后可能崩溃，导致 WSL 实例意外终止。

**恢复**：Windows PowerShell 执行 `wsl --shutdown`，重新打开终端，重新运行程序。Cookie 保存在 `cookie.txt` 中，无需重新登录。

### Cookie 过期？

程序自动检测：访问书架时被重定向到登录页 → 清除旧 cookie → 弹出浏览器重新扫码。用户选 `[Y]` 继续或 `[N]` 退出。

### 内容不完整？

- 图片/表格内的文字无法捕获（Canvas Hook 的固有限制）
- 扉页（无章节标题）无法导航，导出从版权信息页开始
- 付费书只能导出试读部分

### 可以同时导出多本书？

**同一账号不建议**——多终端同步翻页模式异常，可能触发风控。不同账号各一个终端安全。

### 导出文件在哪里？

应用版：`output/书名/书名.md`
Skill 版：按一级标题分文件，`output/书名/第X章.md`

可通过 `cp` 复制到 Windows 桌面：`cp output/* /mnt/c/Users/你的用户名/Desktop/`

## 风险提示

### ⚠️ 版权说明

本工具仅供学习、研究和个人合理使用。导出内容仅限个人 AI 分析和知识管理，请勿：

- 将导出的内容公开发布、传播或商用
- 批量下载侵犯版权的内容
- 规避微信读书的付费会员机制

用户应自行承担使用本工具的法律责任。请尊重原作者和出版方的知识产权。

### ⚠️ 账号安全

- **每页翻页间隔约 2 秒**，模拟人类阅读速度
- **不要同时用多个终端操作同一账号**
- 导出完成后及时退出程序，不要长时间保持浏览器连接
- 频繁、高速翻页可能触发微信读书的风控机制，导致账号受限
- 使用虚拟环境运行，避免依赖冲突

### ⚠️ 数据安全

- `cookie.txt` 包含登录凭据，**请勿分享给他人**
- 导出内容为纯文本 Markdown，不含任何用户身份信息
- 如使用后台运行模式，注意保护好 `nohup` 输出的日志文件

### ⚠️ 技术局限性

- Canvas Hook 无法捕获图片/表格内的文字
- WSL 环境长时间运行 Chromium 存在稳定性风险（有过 WSL 崩溃记录）
- 导出速度受微信读书页面渲染速度和翻页间隔限制，无法加速
